# 테스트 자동 생성

파일 또는 컴포넌트에 대한 포괄적 테스트 스위트를 생성합니다.

## 사용 방법

```text
/roto-generate-tests [파일 경로 또는 컴포넌트명]
```

## 워크플로우

### Step 1: 테스트 환경 탐지

프로젝트의 테스트 프레임워크와 설정을 자동 탐지합니다:

- `package.json`에서 테스트 프레임워크 확인 (vitest, jest, mocha 등)
- `vitest.config.*`, `jest.config.*` 등 설정 파일 확인
- 기존 테스트 파일 패턴 확인 (`*.test.*`, `*.spec.*`)
- 테스트 유틸리티/헬퍼 파일 확인

### Step 2: 대상 코드 분석

지정된 파일/컴포넌트를 분석합니다:

- 테스트 가능한 함수, 메서드, 동작 식별
- 외부 의존성 파악 (모킹 대상)
- 입출력 타입 분석
- 엣지 케이스와 에러 시나리오 식별

### Step 3: 테스트 파일 생성

프로젝트 네이밍 컨벤션에 따라 테스트 파일을 생성합니다.

**테스트 유형**:

#### 유닛 테스트
- 개별 함수의 다양한 입력값 테스트
- 컴포넌트 렌더링과 props 핸들링
- 상태 관리와 라이프사이클
- 유틸리티 함수의 엣지 케이스와 에러 조건

#### 통합 테스트
- 컴포넌트 상호작용
- 모킹된 API 응답과의 통합
- 서비스 레이어 통합
- E2E 사용자 워크플로우

#### 프레임워크별 테스트
- **React**: React Testing Library 기반
- **Vue**: Vue Test Utils 기반
- **Node.js**: API 엔드포인트 및 미들웨어

### Step 4: 테스트 실행 및 검증

```bash
# 프로젝트의 테스트 명령 실행
npx vitest run [테스트 파일] # 또는 npx jest [테스트 파일]
```

실패하는 테스트가 있으면 수정합니다.

## 테스트 작성 원칙

### 구조
- 동작을 설명하는 명확한 테스트명
- AAA 패턴 (Arrange, Act, Assert)
- describe 블록으로 관련 테스트 그룹화
- 테스트 격리를 위한 setup/teardown

### 모킹 전략
- 외부 의존성과 API 호출 모킹
- 테스트 데이터 팩토리 사용
- 비동기 작업의 적절한 정리
- 결정론적 테스트를 위한 타이머/날짜 모킹

### 커버리지 목표
- 80%+ 코드 커버리지
- 핵심 비즈니스 로직 경로 집중
- 성공 경로와 에러 시나리오 모두 테스트
- 경계값 테스트 포함
