# Deep Analysis and Problem Solving Mode

다차원 분석과 깊은 문제 해결을 수행합니다.

## 사용 방법

```text
/roto-ultra-think [분석할 문제 또는 질문]
```

## 분석 프레임워크

### 1. 문제 파싱
- 핵심 도전 과제 추출
- 이해관계자와 제약 조건 식별
- 암묵적 요구사항과 숨겨진 복잡성 파악
- 가정 질문 및 미지 항목 표면화

### 2. 다차원 분석

**기술적 관점**: 기술적 타당성, 확장성, 성능, 유지보수성, 보안, 기술 부채
**비즈니스 관점**: 비즈니스 가치, ROI, 시장 출시 시간, 경쟁 우위, 리스크 대비 보상
**사용자 관점**: 사용자 니즈, 사용성, 접근성, UX 영향, 엣지 케이스
**시스템 관점**: 시스템 전체 영향, 통합 지점, 의존성, 커플링, 창발적 행동

### 3. 복수 솔루션 생성
- 최소 3~5개의 다른 접근법을 브레인스토밍
- 각 접근법에 대해: 장단점, 구현 복잡도, 리소스 요구, 잠재적 리스크, 장기적 영향
- 기존 솔루션과 창의적 솔루션 모두 포함
- 하이브리드 접근법 고려

### 4. 심층 분석
가장 유망한 솔루션에 대해:
- 상세 구현 계획
- 잠재적 함정과 완화 전략
- 단계적 접근법과 MVP
- 2차, 3차 효과 분석
- 실패 모드와 복구 방안

### 5. 교차 도메인 사고
- 다른 산업/도메인에서의 유사 사례
- 다른 컨텍스트의 디자인 패턴 적용
- 자연 시스템 비유
- 기존 솔루션의 혁신적 조합

### 6. 도전과 검증
- 각 솔루션에 대한 반론
- 약점과 사각지대 식별
- "만약에" 시나리오 고려
- 가정 스트레스 테스트
- 의도치 않은 결과 탐색

### 7. 구조화된 결과 출력

```markdown
## 문제 분석
- 핵심 도전 과제
- 주요 제약 조건
- 핵심 성공 요소

## 솔루션 옵션
### 옵션 1: [이름]
- 설명 / 장단점 / 구현 접근법 / 리스크 평가

### 옵션 2: [이름]
[동일 구조]

## 추천안
- 추천 접근법과 근거
- 구현 로드맵
- 성공 지표
- 리스크 완화 계획

## 추가 고려사항
- 반대 의견
- 향후 고려사항
- 추가 조사 필요 영역
```

## 핵심 원칙

- **제1원칙 사고**: 근본적 진실로 분해
- **시스템 사고**: 상호연결과 피드백 루프 고려
- **확률적 사고**: 불확실성과 범위를 다루기
- **역발상**: 무엇을 피해야 하는지도 고려
- **2차 효과 사고**: 결과의 결과를 고려
